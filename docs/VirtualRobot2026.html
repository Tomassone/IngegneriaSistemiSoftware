VirtualRobot2026

Introduction

Goal: Costruire un sistema software che comanda un Differential Drive Robot (DDR).

Requirements

Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota. Il DDR si deve fermare quando tornato in HOME. Il DDR è anche sensibile ai dati rilevati da un Radar. Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata "DMIN":

  • se il DDR si sta muovendo, il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino
  • se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare

Requirement analysis

Dall'interazione con il committente è stato possibile evincere e descrivere con più precisione i seguenti concetti:

HOME:

In primo luogo, risulterà necessario formalizzare il concetto di stanza, di cui HOME è un sottoinsieme. Si potrà modellare la stanza come un quadrato euclideo di base B e altezza H (entrambe misurate in metri). Dopo aver introdotto una serie di assi cartesiani x e y, si potrà dunque identificare ogni punto della stanza con una coppia di coordinate. Il robot però non occuperà un singolo punto, ma un'intera superficie quadrata di lato R (metri). Risulterà dunque conveniente vedere la stanza come una griglia di celle quadrate di lato R (si dirà dunque che l'unità di misura scelta è l'unità robotica).

Di seguito la formalizzazione in forma di un'interfaccia Java: 

			public interface Coordinates {
				int x;
				int y;

				public int getX();
				public int getY();
				public int setX();
				public int setY();
			}

			public interface Room {
				double H; //altezza della stanza in unità robotiche
				double B; //base della stanza in unità robotiche
				double R; //valore in metri dell'unità robotica
				Coordinates homeCoord;

				public double getRoomHeight(); //in metri
				public double getRoomWidth(); //in metri
				public Coordinates getHomeCoordinates();
				public void setRoomHeight(double val); //in metri
				public void setRoomWidth(double val); //in metri
				public void setHomeCoordinates(Coordinates val);
				public boolean isTraversable(Coordinates val); /*per sapere se una
																 cella è occupata*/
				public void setTraversable(Coordinates val);
			}

Radar:

Si tratterà di un'entità che riconosce e comunica regolarmente l'eventuale presenza di ostacoli nelle vicinanze del robot, con un range di riconoscimento di massimo DMIN metri. Per via della natura reattiva del radar, è ragionevole pensare che lo si possa formalizzare con un QActor. Ragionando, è possibile intuire come vi siano due modelli possibili di funzionamento:

  • tramite l'emissione regolare, da parte del radar, di un evento distance(D), che verrà ricevuto da ogni entità del sistema (non essendo al radar noto il destinatario dell'informazione);
  • tramite una risposta, da parte del radar, ad una richiesta getDistance(D) da parte del sistema.

Di seguito la formalizzazione in forma di codice: 

			System ddrsys
			
			Event radarEvent : distance(D)
			Context ctxddr ip [ host="localhost" port=8070 ]

			QActor radar context ctxddr {
				State s0 initial {
					/* inizializzazione stato radar */
				}
				Goto mesureDistance
				
				State mesureDistance{
					emit radarEvent : distance(D)
				}
				Transition t0 
					whenTimeVar Timer -> mesureDistance
			}
		
			System ddrsys
			
			Request getDistance : getDistance(D)
			Reply radar : distance(D) for getDistance

			Context ctxddr ip [ host="localhost" port=8070 ]

			QActor radar context ctxddrsys {
				State s0 initial {
					/* inizializzazione stato radar */
				}
				Goto mesureDistance
				
				State mesureDistance{
					/* rilevamento distanza D */
				}
				Transition t0 
					whenTimeVar Timer -> mesureDistance
					whenRequest getDistance -> answerRequest
			
				State answerRequest{
					onMsg( getDistance : getDistance(D) ){
						replyTo getDistance with radar : distance( $D )  
					} 
				}  
				Transition t0 
					whenTimeVar Timer -> mesureDistance
			}

DDR:

Entità passiva capace di attuare comandi. Può muoversi in avanti o indietro e di ruotare su se stesso di 90° in senso orario o antiorario. La realizzazione di esso è già fornita dal committente.

Mind:

Entità proattiva e reattiva (da qui l'idea di modellarla con un QActor) responsabile di fornire comandi al DDR, in accordo al tipo di comportamento richiesto ed espresso nei requisiti. Chiaramente a tal fine esso dovrà gestire la comunicazione con il radar.

Di seguito la formalizzazione in forma di codice: 

			QActor mind context ctxddr {
			State s0 initial {
				/* inizializzazione stato mente */
			}
			Goto work
			State work {
				/* muove robot sul perimetro della stanza, 
				   utilizzando i metodi esposti */
			}
			Transition t0
				whenEvent radarEvent -> handleRadar
				
			State handleRadar {
				onMsg(radar : distance(D)) {
					[# 
						var D = payloadArg(0).toLong()
					#]
					/* implementazione del comportamento del DDR (dipendente
					   dal valore di D) descritto nei requisiti */
				}
			}
			Goto s0
		}
